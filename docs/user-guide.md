## 2. 三种检测路线

### 2.1 路线一：人工标注 + 监督训练

**适用场景**: 缺陷类别明确，NG样本相对充足，需要分类检出。

**操作流程:**

```
步骤1: 创建项目
  项目管理 → + 创建项目 → 选"缺陷检测" → 命名

步骤2: 上传图片
  进入项目 → 图片管理 → + 上传图片 → 批量选择

步骤3: 标注图片
  图片管理 → 点击图片的"标注"按钮 → 进入标注画布

  标注工具:
  ┌─────────┬─────────────────────────────────┐
  │ BBox模式 │ 在图上拖拽绘制矩形框             │
  │ Polygon  │ 点击添加顶点，双击完成闭合       │
  │ OK/NG    │ 整图级别标签                     │
  │ 类别     │ scratch / dent / stain / crack... │
  └─────────┴─────────────────────────────────┘

  每一张图标注完成后点击"保存标注"

步骤4: 创建数据集
  训练页 → 快速创建并冻结数据集
  系统自动按 70%/15%/15% 切分为 train/val/test

步骤5: 启动训练
  训练页 → 选数据集 → 选任务类型 → 启动训练
  状态: queued → running → succeeded

步骤6: 评估模型
  评估页 → 选模型 → 运行评估
  查看 Precision/Recall/F1/误报/漏报/失败样本

步骤7: 管理模型
  模型页 → 在线测试 → 导出 → 部署 → 回滚
```

### 2.2 路线二：AI 预标注 + 人工审核

**适用场景**: 图片量大，希望AI先跑一遍，人工只需确认。

**操作流程:**

```
步骤1-2: 同上（创建项目、上传图片）

步骤3: 启动AI预标注
  图片管理 → AI预标注(批量) → 系统自动生成候选标注

  ⚠️ 当前版本: AI预标注为 mock 数据（随机生成候选框）
     后续可接入 GroundingDINO / SAM 等真实模型

步骤4: 审核候选标注
  进入项目 → 审核队列
  或: 图片管理 → 点击图片标注 → 右侧面板有黄色候选标注

  对每个候选标注:
  ✅ 确认 → 变为正式标注 (绿色)
  ❌ 拒绝 → 删除候选
  ✏️ 修改 → 手动调整后保存

步骤5-7: 同路线一（数据集→训练→评估→模型）
```

### 2.3 路线三：OK 样本异常检测

**适用场景**: NG样本极少或没有，只上传合格品图片训练"正常"模型。

**操作流程:**

```
步骤1: 创建项目 → 选择"异常检测"类型

步骤2: 上传OK图片 → 只上传无缺陷的正常样本

步骤3: 训练异常模型
  进入项目 → 异常检测 → 训练模型
  系统学习OK样本的特征分布

步骤4: 测试
  上传测试图片 → 系统输出:
  - anomaly score (异常分数)
  - OK/NG 判定
  - 阈值可调

步骤5: 评估
  查看 AUROC / AUPR / 误报率
```

## 3. 核心功能详解

### 3.1 标注画布

```
┌────────┬──────────────────────┬──────────┐
│ 工具区  │      Canvas画布       │ 标注列表  │
│        │                      │          │
│ BBox   │   [图片 + 标注框]     │ scratch  │
│ Polygon│                      │ dent x2  │
│ OK/NG  │   拖拽或点击绘制      │ stain    │
│ 类别   │                      │ ...      │
│        │                      │          │
│ 上一张  │                      │ 审核按钮  │
│ 下一张  │                      │          │
└────────┴──────────────────────┴──────────┘
```

**快捷键**: 
- BBox模式: 拖拽绘制
- Polygon模式: 点击添加顶点, 双击完成
- 切换类目: 点击左侧标签

### 3.2 审核队列

`项目 → 审核队列` 页面可批量处理AI预标注候选:

- **全选**: 选中所有待审核标注
- **批量确认**: 一键确认所有选中项
- **批量拒绝**: 一键拒绝所有选中项
- **单图处理**: 逐张图片逐个确认/拒绝

### 3.3 训练任务

```
训练页包含:
┌─────────────────────────────────────┐
│ 启动新训练                           │
│  数据集版本 [选择] [快速创建]         │
│  任务类型   [缺陷检测|分类|分割|异常]  │
│  训练策略   [Fast Baseline|高精度]    │
│  [启动训练]                          │
├─────────────────────────────────────┤
│ 训练历史                             │
│  ID | 类型 | 状态 | P/R/F1 | 时间     │
│  ...                                │
└─────────────────────────────────────┘
```

### 3.4 评估报告

- **指标卡片**: Precision / Recall / F1 / 推理耗时
- **图表**: 柱状图 + 雷达图
- **误报/漏报**: FP/FN计数
- **失败样本**: Top错误案例
- **阈值调节**: 0-1 滑块实时调整
- **导出**: JSON + Markdown 两种格式

### 3.5 模型管理

| 操作 | 说明 |
|---|---|
| 在线测试 | Mock推理，返回预测框+置信度 |
| 导出模型 | 导出ONNX格式 |
| 部署 | 创建推理端点 |
| 回滚 | 回到上一版本 |
| 对比 | 多模型指标柱状图对比 |

### 3.6 License管理

`License` 页面查看:
- 套餐类型和有效期
- 用量进度条（项目/图片/训练/存储）
- 限制详情表

## 5. AI Mock 说明

**MVP阶段所有AI功能均为mock（模拟数据）:**

| 功能 | Mock行为 | 替换方式 |
|---|---|---|
| **AI预标注** | 随机生成BBox (确定性随机, 同图同结果) | 替换 `app/ai/mock_prelabel.py` |
| **训练** | 模拟5轮训练, 生成合理指标 | 替换 `app/ai/mock_training.py` |
| **评估** | 随机生成 Precision/Recall/F1/失败样本 | 替换 `app/ai/mock_evaluation.py` |
| **在线推理** | 随机返回预测框 | 接入 ONNX Runtime / TensorRT |

**Mock的接口格式是真实的**，后续接入真实算法模型时，只需要替换 `app/ai/` 目录下的Python文件，保持相同的输入输出格式即可，无需修改任何业务代码。
