# AI 大模型集成路线

> 当前状态：OpenCV + sklearn（CPU可跑，无需GPU）
> 目标：SAM 分割 → Grounding DINO 检测 → VLM 判断，三阶段递进

---

## 一、SAM2 — 点击分割

### 做什么

用户在图片上点一下缺陷区域，SAM 自动生成精确多边形 mask，替代手动画 polygon。

### 对比

| | 手动画 Polygon | SAM 点击分割 |
|---|---|---|
| 操作 | 逐点点击，n次顶点，闭合 | 点一下 |
| 精度 | 人眼估计 | 像素级 mask |
| 速度 | 15-30s/个 | 2s/个 |
| 学习成本 | 需要训练 | 零 |

### 技术选型

**模型**: facebookresearch/sam2  
**大小**: 93MB (tiny), 2.4GB (large)  
**推理**: CPU 15s, T4 GPU 2s, A10 GPU 0.5s

### 集成方案

```
阶段1 (编码): 图片 → SAM ViT → 存特征 (每图一次, 170MB/图)
阶段2 (解码): 用户点击坐标 → 特征+点提示 → SAM解码 → mask轮廓 → polygon存入
```

**API设计**:
```python
POST /api/images/{id}/sam/encode     # 预处理，缓存特征
POST /api/images/{id}/sam/decode     # {x, y} → 返回 polygon geometry
```

**前端改动**:
- 标注页新增"SAM分割"按钮
- 点击模式：用户点一下 → API返回多边形 → 画布渲染

### 改动量

| 组件 | 工作 |
|---|---|
| `app/ai/sam_encoder.py` | 加载模型, 编码图片, 缓存特征 |
| `app/ai/sam_decoder.py` | 坐标 → mask → polygon |
| `app/routers/sam.py` | 2个 API 端点 |
| 前端 AnnotationPage | 加按钮 + 点击回调 |
| requirements.txt | + segment-anything |

**工作量**: ~4小时

### 成本

- 本地部署: 需要 GPU（T4 以上），模型下载 93MB-2.4GB
- 云端 API: 无现成服务，需自建

## 二、Grounding DINO + SAM — 文字提示检测

### 做什么

用户输入"找出所有划痕"，模型自动检测所有划痕并生成分割 mask。

### 流程

```
文字提示 ("find scratches")
  → Grounding DINO (找所有 bbox)
  → SAM (逐框分割)
  → 输出 polygon 列表 → 存为 candidate annotations
```

### 技术选型

**Grounding DINO**: IDEA-Research/GroundingDINO  
**大小**: 240MB (base), 680MB (large)  
**推理**: T4 GPU 3-5s/图

### 与当前 OpenCV 预标注对比

| | OpenCV | Grounding DINO + SAM |
|---|---|---|
| 准确率 | ~40% (Canny 噪声大) | ~80%+ |
| 支持文字提示 | 不支持 | 支持 |
| 输出格式 | BBox | BBox + Polygon Mask |
| 硬件需求 | CPU | GPU (T4) |
| 推理速度 | 0.5s | 3-5s |

### 集成方案

**替换**: `app/ai/real_prelabel.py` 增加 `method="grounding_dino"` 选项

**前端支持**:
- 预标注引擎下拉框新增 "Grounding DINO"
- 可选输入文字提示（默认 "defect, scratch, dent, crack"）

### 改动量

| 组件 | 工作 |
|---|---|
| `app/ai/grounding_dino_prelabel.py` | 加载模型，文字→bbox→SAM分割→annotations |
| `app/ai/real_prelabel.py` | 新增 method="grounding_dino" 分支 |
| 前端 ImageListPage | 下拉加选项 + 可选提示词输入 |
| requirements.txt | + groundingdino-py |

**工作量**: ~6小时

## 三、VLM — 视觉语言模型判断

### 做什么

整图级别的质量判断。GPT-4o / Qwen2-VL 评估图片并输出结构化结果。

### 使用场景

1. **自动初筛**: 上传即自动判断 OK/NG，不合格再进人工审核
2. **缺陷描述**: 不只判断有无，还描述缺陷类型、位置、严重程度
3. **批量报告**: 100张图片 → VLM 逐一分析 → 生成检测报告

### 技术选型

| 模型 | 大小 | 硬件 | 速度 | 成本 | 适用 |
|---|---|---|---|---|---|
| GPT-4o API | - | 无 | 2s | ~¥0.01/图 | 最快上线 |
| Qwen2-VL-7B | 14GB | A10 24G | 3s | 免费(自建) | 私有化 |
| InternVL2-4B | 8GB | T4 16G | 2s | 免费(自建) | 性价比 |
| MiniCPM-V-2.6 | 8GB | T4 16G | 1.5s | 免费(自建) | 最快 |

### API 设计

```python
POST /api/images/{id}/analyze
{
  "model": "gpt-4o",        # 或 "qwen2-vl-7b"
  "prompt": "这张图有缺陷吗？列出所有缺陷的类型、位置和严重程度。"
}
→ {
  "ok": false,
  "confidence": 0.92,
  "defects": [
    {"type": "scratch", "location": "左上角", "severity": "minor"},
    {"type": "dent", "location": "中部偏右", "severity": "major"}
  ],
  "summary": "检测到2处缺陷，建议NG处理"
}
```

### 改动量

| 组件 | 工作 |
|---|---|
| `app/ai/vlm_analyzer.py` | API调用/本地推理 |
| `app/routers/vlm.py` | analyze 端点 |
| 前端 ImageListPage | "VLM分析"按钮 |

### GPT-4o 方案（推荐第一阶段）

```python
import openai
def analyze_image(image_base64, prompt):
    client = openai.OpenAI(api_key=os.getenv("OPENAI_API_KEY"))
    r = client.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": [
            {"type": "text", "text": prompt},
            {"type": "image_url", "image_url": {"url": f"data:image/jpeg;base64,{image_base64}"}}
        ]}]
    )
    return parse_response(r.choices[0].message.content)
```

**零模型部署，只需 API Key。**
