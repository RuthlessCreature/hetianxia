# 部署指南

## 服务器要求

- Linux (Alibaba Linux 3 / Ubuntu 20.04+)
- Docker + Docker Compose
- 内存 >= 1GB
- 端口 80 开放

## 完整流程

### 1. 装 Docker

```bash
# Alibaba Linux 3
dnf config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
dnf install -y docker-ce docker-compose-plugin

# 国内镜像加速
cat > /etc/docker/daemon.json << 'EOF'
{"registry-mirrors":["https://docker.1panel.live","https://hub.rat.dev"]}
EOF
systemctl daemon-reload && systemctl enable --now docker
```

### 2. 拉代码

```bash
cd /root
git clone <仓库地址> hetianxia
cd hetianxia
```

### 3. 配置

```bash
cat > .env << EOF
HTX_APP_NAME=高纳AI
HTX_COMPANY_NAME=高纳科技
HTX_PLATFORM_NAME=工业视觉平台
HTX_THEME=light
HTX_LAYOUT=sidebar
HTX_HOST_BIND=0.0.0.0
HTX_FRONTEND_PORT=80
HTX_BACKEND_PORT=8000
HTX_SAM_MODEL_SIZE=tiny
HTX_SAM_MODELS_DIR=./models
HTX_DATABASE_URL=sqlite:///./data/hetianxia.db
HTX_SECRET_KEY=$(openssl rand -hex 32)
HTX_ALGORITHM=HS256
HTX_ACCESS_TOKEN_EXPIRE_MINUTES=1440
HTX_UPLOAD_DIR=./data/uploads
HTX_THUMBNAIL_DIR=./data/thumbnails
EOF
mkdir -p data/uploads data/thumbnails data/models
```

### 3.1 下载 AI 模型

Ubuntu 一键命令：

```bash
sudo apt-get update && sudo apt-get install -y curl ca-certificates
bash scripts/download-ai-models.sh
```

默认下载 `YOLOv8n` + `SAM2 tiny`。如果要把四个 SAM2 模型一次下全：

```bash
bash scripts/download-ai-models.sh --all
```

模型位置：

| 模型 | 保存位置 |
|---|---|
| YOLOv8n | `backend/yolov8n.pt` |
| SAM2 | `backend/models/*.pt` |

Docker Compose 会把 `backend/models` 挂到后端容器的 `/app/models`。

### 4. 启动

```bash
bash scripts/up.sh
```

### 5. 验证

```
http://服务器IP
```

## 运维命令

```bash
docker-compose ps              # 状态（Compose v1）
docker-compose logs -f         # 日志（Compose v1）
docker-compose restart         # 重启（Compose v1）
docker-compose down            # 停止（Compose v1）
bash scripts/up.sh             # 自动识别 Compose v1/v2 并启动
```

## 升级

```bash
git pull
bash scripts/up.sh
```

## 改名字

编辑 `.env`:
```
HTX_APP_NAME=你的产品名
HTX_COMPANY_NAME=你的公司名
HTX_PLATFORM_NAME=你的平台名
HTX_THEME=dark
HTX_LAYOUT=topbar
HTX_HOST_BIND=0.0.0.0
HTX_FRONTEND_PORT=8080
HTX_BACKEND_PORT=18000
```
然后 `bash scripts/up.sh`。这些分发参数由前端容器启动时生成的 `/config.js` 注入，通常不需要重新构建镜像。

也可以在启动命令里直接覆盖：

```bash
HTX_APP_NAME=锐检AI HTX_COMPANY_NAME=锐检科技 HTX_PLATFORM_NAME=缺陷检测平台 HTX_THEME=industrial HTX_LAYOUT=focus HTX_HOST_BIND=0.0.0.0 HTX_FRONTEND_PORT=8080 HTX_BACKEND_PORT=18000 bash scripts/up.sh
```
