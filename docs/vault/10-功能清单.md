## 1. 多租户与权限

- 租户隔离：所有数据按 tenant_id 过滤
- 套餐体系：Trial / Basic / Pro / Enterprise

| Owner | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |

| Engineer | ✅ | ✅ | ✅ | ✅ | ❌ | ❌ |
| Labeler | ✅ | ✅ | ✅ | ❌ | ❌ | ❌ |

- 用量限制：项目数、图片数、存储、月训练次数、月预标注次数

## 2. 主题与品牌

### 2.1 可配置品牌

| 配置项 | `.env` 变量 | 说明 |
|---|---|---|
| 软件名称 | `HTX_APP_NAME` | 登录页、侧边栏、标题栏 |
| 公司名称 | `HTX_COMPANY_NAME` | API / 后端标识 |
| 平台名称 | `HTX_PLATFORM_NAME` | 登录页副标题、浏览器标题、布局品牌区 |
| 界面主题 | `HTX_THEME` | 三选一 |
| 界面布局 | `HTX_LAYOUT` | 三选一 |
| 监听地址 | `HTX_HOST_BIND` | 宿主机绑定地址 |
| 前端端口 | `HTX_FRONTEND_PORT` | 前端宿主机端口 |
| 后端端口 | `HTX_BACKEND_PORT` | 后端 API 宿主机端口 |
| SAM2 默认模型 | `HTX_SAM_MODEL_SIZE` | tiny / small / base / large |
| SAM2 模型目录 | `HTX_SAM_MODELS_DIR` | 容器内模型目录 |

这些参数支持通过 `.env` 或启动命令注入；前端生产镜像会在容器启动时生成 `/config.js`，只换分发参数不需要重新构建镜像。

### 2.2 三套主题

| 主题 | 风格 | 适用场景 |
|---|---|---|
| `light` | 白底 zinc 侧边栏 | 默认，通用 |
| `dark` | 黑底白字 | 暗光环境、偏好 |
| `industrial` | 深绿底 teal + 等宽字体 | AOI 产线终端 |

### 2.3 三套布局

| 布局 | 风格 | 适用场景 |
|---|---|---|
| `sidebar` | 左侧导航 + 顶栏 | 默认后台管理 |
| `topbar` | 顶部品牌栏 + 横向导航 | 白标客户、宽屏演示 |
| `focus` | 窄图标栏 + 工作区 | 产线终端、标注/审核聚焦 |

### 2.4 移动端与 Pad 适配

- 手机端统一切换到底部导航，避免左侧栏挤占工作区
- Pad 端保留对应分发布局：`sidebar` 左栏、`topbar` 顶栏、`focus` 窄图标栏
- 表格、顶部 tab、筛选工具条、图片网格和标注工作台都有横向滚动或断点重排兜底
- 三套主题共用响应式结构，颜色变量继续由 `HTX_THEME` 控制

## 5. 标注系统

### 5.1 标注类型

| 类型 | 说明 | 几何数据 |
|---|---|---|
| BBox | 矩形框 | `{x, y, w, h}` |
| Polygon | 多边形 | `{points: [{x,y}, ...]}` |
| Image Label | 整图标签 | null |

### 5.2 标注画布 (Fabric.js)

- **框模式**: 拖拽空白画布 → 松手保存 BBox。点已有框选中，不产生新框。
- **多边形模式**: 点击空白加顶点 → 点"闭合"按钮或双击完成。顶点数实时显示。
- **选择模式**: 点标注选中高亮 → 拖拽移动 → Delete 删除。右侧下拉改标签。
- **标签跟框联动**: BBox 和文字标签是 Fabric.js Group，拖拽框时标签跟随移动。
- **标注列表**: 右侧面板显示全部标注，点击列表项选中画布对应框。选中标注显示浮动详情（可改标签、删除）。
- **上下翻页**: 上一张/下一张按钮，显示当前序号/总数。
- **自定义标签**: 左侧标签列表底部输入框，添加自定义缺陷类型名。localStorage 持久化。

### 5.3 审核

- 候选标注 → 橙色虚线框
- 审核队列独立页面：按图片分组展示，批量确认/拒绝
- 标注页内审核：右侧面板黄色区域，确认→变绿实线，拒绝→移除
- 拒绝后标注数实时递减

### 5.4 标注来源

| 来源 | 说明 |
|---|---|
| `human` | 人工标注 |
| `ai_prelabel` | AI 预标注（需审核才能进训练集） |
| `model_inference` | 模型推理（预留） |
| `imported` | 外部导入（预留） |

### 5.5 标注状态

```
raw → prelabeled(candidate) → confirmed / rejected
     ↘ labeled(human direct) → confirmed
```

## 6. AI 预标注

### 6.1 当前引擎

**混合模式 (hybrid, 默认)**:
- YOLOv8-nano (6MB) → 通用目标检测，低阈值 0.05（需要 torch，可选）
- OpenCV Canny 边缘 → 划痕/裂纹识别
- OpenCV DoG 斑点 → 凹坑/污渍识别
- OpenCV Harris 角点 → 边缘毛刺识别
- 结果合并去重 → 输出 candidate annotations

**OpenCV only**: 纯传统 CV，不需要 PyTorch

### 6.2 坐标保护

- 后端检测结果强制钳位到图像边界
- 前端渲染防御性二次钳位
- 面积 >70% 的超大框自动丢弃
- 十字标记/参考标记自动过滤

### 6.3 计划接入

| 引擎 | 说明 | 状态 |
|---|---|---|
| SAM2 | 点击→自动分割 mask | 规划中 |
| Grounding DINO | 文字提示→检测+分割 | 规划中 |
| VLM | 整图 OK/NG 判断 | 规划中 |

## 7. 数据集管理

- 创建数据集版本（v1, v2, v3...）
- 冻结：从项目中已标注图片自动按比例切分 train/val/test（默认 70/15/15）
- 冻结后不可修改，只能创建新版本
- 训练任务引用 dataset_version

---

## 8. 训练

### 8.1 当前算法

| 环境 | 算法 | 说明 |
|---|---|---|
| 无 PyTorch (默认) | sklearn RandomForest + 18维手工特征 | CPU 运行，秒级完成 |
| 有 PyTorch | ResNet-18/34/50 + EfficientNet-B0 + 迁移学习 | GPU 推荐 |

### 8.2 流程

1. 选择数据集版本（冻结后）
2. 选择任务类型（缺陷检测/分类/分割/异常检测）
3. 选择模型/策略
4. 启动训练 → queued → running → succeeded → 自动创建 ModelRegistry

### 8.3 训练完成后

- 自动创建模型注册表记录
- 自动发通知（训练完成，含指标摘要）
- 自动写审计日志

## 9. 评估

- 选择模型版本 + 测试集
- 阈值滑块 (0.00-1.00) 实时调节
- 指标卡片: Precision / Recall / F1 / 推理耗时
- Recharts 柱状图 + 雷达图
- 误报/漏报统计 (FP / FN)
- 失败样本 Top N 列表
- 导出 JSON 报告
- 导出 Markdown 报告（含指标表格+失败样本）
- 评估历史列表

---

## 10. 模型管理

- 模型版本列表（自动版本号 M-2026-001）
- 模型详情卡片（Precision/Recall/F1 三色仪表）
- **在线测试**: Mock 推理，返回预测框 + 置信度
- **导出**: ONNX/pkl 格式
- **部署**: 创建推理端点
- **回滚**: 回到上一版本
- **对比**: 多选模型，柱状图对比 P/R/F1/mAP

---

## 11. 异常检测（独立路线）

- 专属项目类型 `anomaly_detection`
- 三步骤向导：上传OK图片 → 训练正常分布模型 → 测试
- 马氏距离（高斯分布）计算异常分数
- 阈值判定 OK/NG
- 指标: AUROC / AUPR

## 13. 安全

- JWT 认证（token 含 user_id + tenant_id）
- 租户隔离（所有 API 查询自动带 tenant_id 过滤）
- RBAC 角色权限强制拦截
- License 到期拦截（训练/上传等写操作）
- 用量超限拦截（项目数/图片数/训练次数）
- API 限流 (120次/分钟/IP)，返回 X-RateLimit-Remaining
- AI 预标注 candidate ≠ ground truth（必须人工确认才能进训练集）
