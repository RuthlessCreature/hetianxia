# 主题与分发布局配置

同一套底层功能支持多套分发外壳。启动时通过环境变量切换品牌、平台名、主题和布局。

## 品牌参数

| 变量 | 说明 | 默认值 |
|---|---|---|
| `HTX_APP_NAME` | 产品/软件名称 | 高纳AI |
| `HTX_COMPANY_NAME` | 公司名称 | 高纳科技 |
| `HTX_PLATFORM_NAME` | 平台名称/副标题 | 工业视觉平台 |

Docker 启动示例：

```bash
HTX_APP_NAME=锐检AI HTX_COMPANY_NAME=锐检科技 HTX_PLATFORM_NAME=缺陷检测平台 docker compose up -d
```

## light

默认。白底灰字，zinc-950 深色侧边栏。

```env
HTX_THEME=light
```

## dark

黑底白字，全暗模式。

```env
HTX_THEME=dark
```

## industrial

AOI 风格。深绿底 + teal 强调色 + 等宽字体。

```env
HTX_THEME=industrial
```

## 布局

| 布局 | 风格 | 适用场景 |
|---|---|---|
| `sidebar` | 左侧导航 + 顶栏 | 默认后台管理 |
| `topbar` | 顶部品牌栏 + 横向导航 | 白标客户、宽屏演示 |
| `focus` | 窄图标栏 + 工作区 | 产线终端、标注/审核聚焦 |

```env
HTX_LAYOUT=topbar
```

三套布局都会在手机端自动切换到底部导航，Pad 端保留侧栏/窄栏/顶栏。三套主题共享同一套响应式规则。

## 监听地址与端口

| 变量 | 说明 | 默认值 |
|---|---|---|
| `HTX_HOST_BIND` | 宿主机监听地址 | 0.0.0.0 |
| `HTX_FRONTEND_PORT` | 前端宿主机端口 | 80 |
| `HTX_BACKEND_PORT` | 后端 API 宿主机端口 | 8000 |

```env
HTX_HOST_BIND=0.0.0.0
HTX_FRONTEND_PORT=8080
HTX_BACKEND_PORT=18000
```

## AI 模型参数

| 变量 | 说明 | 默认值 |
|---|---|---|
| `HTX_SAM_MODEL_SIZE` | 默认加载的 SAM2 模型 | tiny |
| `HTX_SAM_MODELS_DIR` | 容器内 SAM2 模型目录 | ./models |

Ubuntu 下载命令：

```bash
bash scripts/download-ai-models.sh
```

## 切换方式

1. 改 `.env` 里的 `HTX_APP_NAME` / `HTX_COMPANY_NAME` / `HTX_PLATFORM_NAME` / `HTX_THEME` / `HTX_LAYOUT` / `HTX_HOST_BIND` / `HTX_FRONTEND_PORT` / `HTX_BACKEND_PORT` / `HTX_SAM_MODEL_SIZE` / `HTX_SAM_MODELS_DIR`
2. `docker compose up -d`
3. 刷新浏览器

前端生产镜像会在容器启动时生成 `/config.js`，所以只改这些分发参数不需要重新构建镜像。代码升级或依赖变化时才需要 `docker compose up -d --build`。

## 自定义主题

编辑 `frontend/src/index.css`，添加新的 `[data-theme="xxx"]` 块。CSS 变量列表：

| 变量 | 说明 |
|---|---|
| --bg-main | 页面背景 |
| --bg-card | 卡片背景 |
| --bg-sidebar | 侧边栏背景 |
| --text-primary | 主文字 |
| --text-secondary | 次文字 |
| --border | 边框颜色 |
| --accent | 强调色(按钮、链接) |
| --header-bg | 顶栏背景 |
| --input-bg | 输入框背景 |
